这是一个将策划配置的Excel表数据转为程序使用的json格式，并提供json数据的加载。加载数据语言暂时支持go即可。


excel表格中的sheet分为两种类型：
1.以@Type定义的为表结构定义sheet
2.其他类型为表数据sheet



表结构定义sheet结构：
第一行为固定（种类	对象类型	中文描述	字段名	字段类型	数组切割	默认值	筛选）
下面是具体举例
种类：
	表头：表的具体表名
	枚举：go 自定义枚举
	结构：不是一个表，而是表内嵌的自定义结构

对象类型：
	和种类关联，如种类是表头，则代表表名，如枚举 则表示自定义的枚举类型名字，如结构则表示结构名
	中文描述：则是一个注释
	字段名：和种类相关，如表头，则表示表中的列名，如枚举 具体的枚举名，如结构则表示结构的字段名
	字段类型：需要支持  string  int   int64 
	数组切割：有值则表示该字段为数组，转表时读取表数据，用该符号将一个单元格的数据切割数组
	默认值：对枚举有效，表示枚举值是多少
	筛选：分为  C  只有客户端到处该字段  S只有服务器到处该字段   CS 服务器客户端都要导出
	
	
	
表数据sheet结构：
	sheet名字为表名字
	sheet第一行第一列为ArrayDict 为固定结构，用于表的索引ID
	第一行后面的N列为表的列名
	对于以#号开头的的列或者行，表示该行或列不参与转表 读取时自动忽略
	第二行开始为表的具体数据内容
	
	
	
没有生成go加载json的代码

不需要输出__enums__.json文件

项目支持配置文件 文件内容如下
{
	inputs:[
		"路径1",
		"路径2",
		"文件1",
		"文件2",
	]
	"output":{
		"codePath": "go代码输出路径"
		"jsonPath": "json文件输出路径"
	}
}

在main开头使用flag加载的参数，全部使用config来进行输入，启动时只需要制定config路径即可


表头中应该有一个默认的ID字段，用作该行数据的唯一ID，不可重复。
并且对于生成的go struct 代码中的所有字段都应该是小写非导出的，避免外部意外修改字段，对于每个字段提供Getxxx  读取该字段数据


每张表提供一个foreach方法遍历表


ID默认应该是int64类型，也可以通过表结构定义sheet中进行修改
